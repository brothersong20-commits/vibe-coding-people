"오늘 뭐 먹지?"는 매일 반복되는 메뉴 선택의 피로를 줄이기 위해 만든 가벼운 사이드 프로젝트입니다. 큰 화면에서 한 번에 결과를 보여주는 것에 집중했습니다.

## 만든 이유

점심 시간마다 "뭐 먹지" 한 마디로 5분이 사라집니다. 골라주는 도구는 많지만 대부분 너무 무거웠습니다. 한 번에 결정을 끝내는 가장 가벼운 형태를 만들어보고 싶었습니다.

## 핵심 흐름

1. 버튼을 한 번 누른다.
2. 결과 한 개가 즉시 뜬다.
3. 마음에 들지 않으면 다시 한 번 누른다.

이 세 줄을 유지하기 위해 다른 모든 옵션을 자제했습니다.

## 사용한 도구

```
Next.js (App Router)
Tailwind CSS
Vercel (배포)
```

## 회고

- 처음에는 카테고리 필터, 가격대, 위치 등 여러 옵션을 넣으려다 결국 모두 뺐습니다. "한 번에 끝나는 도구"라는 결정이 사이트의 정체성이 되었습니다.
- 작은 프로젝트일수록 배포까지 가는 사이클이 짧아야 합니다. **Vercel + git push** 워크플로가 결정적이었습니다.
