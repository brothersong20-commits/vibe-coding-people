- **콘텐츠 관리**는 1단계에서는 저장소 안 TypeScript/MDX로 두고, 추후 Supabase로 옮길 수 있는 데이터 구조를 유지합니다.
- **디자인**은 `DESIGN.md`의 Cursor 마케팅 톤을 그대로 따릅니다. 다크 모드 대신 워밍 크림 캔버스를 사용합니다.
- **단계 로드맵**: 1단계 로컬 → 2단계 Git + Vercel 배포 → 3단계 콘텐츠 확장 → 4단계 Supabase + Google Auth.

## 사용한 도구

```
Next.js (App Router) + TypeScript
Tailwind CSS v4
@next/mdx
date-fns
```
