30일간 AI 와 페어로 사이드 프로젝트를 하나 끝까지 만들어 봤다. 짧은 회고를 남긴다.

## 바뀐 습관

- 코드를 쓰기 전에 **데이터 스키마부터 그린다.** 화면은 그 다음이다.
- 디자인 토큰을 먼저 정한 뒤 화면을 만든다. 컴포넌트 하나 만들 때마다 색상을 고민하던 시간이 사라졌다.
- 작업의 시작과 끝에 **항상 plan 한 줄을 남긴다.** 다음 세션의 나에게 보내는 메모.

## 결과

```
커밋 87개 · PR 12개 · 배포 9회
```

숫자만 보면 평범하지만, **모두 작은 단위**라는 게 핵심이다. 1주차에는 한 PR 에 너무 많은 변경을 욱여넣어 리뷰가 멈췄다. 2주차부터 한 PR = 한 가지 결정으로 묶고 나니, 작업이 멈추지 않는다.

## 후회되는 것

테스트 자동화를 더 일찍 붙였어야 했다. 데모용 토이라 미뤘는데, 라우트가 5개를 넘기는 순간부터는 회귀를 손으로 잡기 어려워졌다. 다음 사이드 프로젝트에서는 1주차에 e2e 한 줄이라도 깔고 시작한다.
