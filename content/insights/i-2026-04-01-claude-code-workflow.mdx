바이브 코딩으로 사이드 프로젝트를 만들면서 가장 크게 바뀐 점은 **plan을 먼저 쓰는 습관**이다. 코드를 곧장 두드리는 대신 의도를 한 번 문장으로 적어 두니 결과물의 결이 매번 또렷해졌다.

## plan-mode 가 가져온 변화

plan-mode 에서는 코드를 직접 고치지 않고, 어떤 파일에 어떤 변경을 줄지 먼저 합의한다. 처음에는 답답했지만 한 주만 굴려도 차이가 보였다.

- 작은 단위로 쪼개진 PR 이 자연스러워졌다.
- 머릿속에서만 굴리던 가설이 텍스트로 남는다.
- 다음 작업을 시작할 때 "지난번 결정"이 그대로 자산이 된다.

## 자동 verify 루프

빌드/타입 체크/lint 를 작업 마지막에 자동으로 돌리도록 훅을 잡았다. 매번 손으로 치는 부담이 사라지자, 작은 변경도 부담 없이 커밋하게 됐다.

```bash
pnpm typecheck && pnpm lint && pnpm build
```
