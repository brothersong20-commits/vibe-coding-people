같은 저장소를 두 에이전트가 본다는 건 생각보다 깔끔하지 않다. 한 달간 Cursor 와 Claude Code 를 한 프로젝트에 같이 붙여 쓰면서 충돌을 줄이는 규칙을 정리했다.

## 역할 분담

- **Cursor**: 파일 단위 인라인 편집, 빠른 자동완성, IDE 안에서 짧은 호흡으로 굴리는 변경.
- **Claude Code**: 라우트·데이터 모델 단위의 큰 변경, plan 작성, 멀티 파일 리팩터링.

도구별 강점을 인정하고 나니, 어디서 어느 쪽을 켜야 할지가 자연스럽게 보였다.

## 충돌을 줄이는 규칙

같은 파일을 두 도구가 동시에 만지지 않도록 한 가지만 지킨다.

```text
PLAN/ 가 열려 있는 동안에는 Cursor 자동 적용을 끈다.
```

큰 변경을 plan-mode 로 정리하는 시점에는 IDE 자동 편집이 의도를 흐리기 쉽다. 결정이 끝난 다음에야 Cursor 의 자동 완성을 다시 켠다.

## 한계

자동 완성 품질은 Cursor 가, 큰 그림은 Claude Code 가 낫다는 결론. 둘 중 하나를 고르기보다, **턴 단위로 도구를 분리해서 쓰는 쪽**이 1단계 사이드 프로젝트에는 잘 맞았다.
