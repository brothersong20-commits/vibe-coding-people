# Phase 3 — `/insights` 라우트 추가

## Context

- 3단계 로드맵의 남은 작업 중 **`/insights` 라우트**를 신규로 추가한다.
- 모임의 **바이브 코딩 학습 인사이트**(AI 코딩 툴 사용기·워크플로우 회고·도구 비교 등 짧은 글)를 외부에 공유하는 허브.
- 같은 단계의 다른 콘텐츠(`/projects`, `/contests`)와 동일한 패턴(`content/*.ts` 메타 + `content/{slug}.mdx` 본문 + 리스트/상세 페이지 + body map)을 그대로 따른다.
- 4단계 Supabase 이관 시 메타가 그대로 테이블 row가 되도록 `id`/`slug`/`date` 등 필드 모양을 미리 정렬한다.

## 결정사항

| 항목 | 결정 |
| --- | --- |
| 페이지 | 리스트 `/insights` + 상세 `/insights/[slug]` (MDX 본문) |
| 카드 레이아웃 | ProjectCard 같은 **그리드 카드** (`md:grid-cols-2 lg:grid-cols-3`) |
| 더미 콘텐츠 | 3건 (AI 코딩 / 도구 / 회고 각 1건) |
| 메타 필드 | `id`, `slug`, `title`, `date`, `category`, `summary`, `tags[]`, `author`, `readTime`(분) |
| 카테고리 enum | `"AI 코딩" \| "도구" \| "워크플로우" \| "회고"` |
| 정렬 | `date` 내림차순 (동일 날짜는 `id` 역순 안정 정렬) |
| TopNav 위치 | `프로젝트` 다음, `공모전` 앞 |

## 데이터 모델

```ts
// content/insights.ts
export type InsightCategory = "AI 코딩" | "도구" | "워크플로우" | "회고";

export type Insight = {
  id: string;        // slug와 동일하게 둔다 (4단계 Supabase에서 PK로 승격)
  slug: string;
  title: string;
  date: string;      // YYYY-MM-DD
  category: InsightCategory;
  summary: string;   // 1줄 — 카드 본문 + meta description
  tags: string[];
  author: string;
  readTime: number;  // 분
};
```

더미 3건:
- `i-2026-04-01-claude-code-workflow` — "AI 코딩" — "Claude Code로 사이드 프로젝트 워크플로우 정착하기"
- `i-2026-04-22-cursor-vs-claude` — "도구" — "Cursor와 Claude Code, 어떻게 같이 쓰면 좋을까"
- `i-2026-05-10-vibe-coding-retrospective` — "회고" — "바이브 코딩 한 달, 무엇이 바뀌었나"

## 컴포넌트 사양

### `InsightCard`
- 외형: `rounded-[var(--radius-lg)]` + 1px `--color-hairline` 보더, hover 시 `--color-hairline-strong`
- 상단: 카테고리 뱃지(`categoryTone` 매핑) + `code` 날짜·`{readTime}분`
- 본문: `title-md` 제목 + `body-md text-[var(--color-body)]` 요약
- 하단: 상단 hairline 구분선 + `code by {author}` + 태그 2개(`Badge variant="soft"`)
- 전체 카드 `<Link href={`/insights/${slug}`}>` 래핑, `group-hover:underline` 제목

카테고리 톤 (단일 매핑, 리스트/상세에서 재사용):
- `AI 코딩` → `--color-primary` 배경
- `도구` → `--color-semantic-success` 배경
- `워크플로우` → `--color-timeline-edit` 배경 (제품 외부 표면이라 톤 차용 OK)
- `회고` → `--color-surface-strong` 배경

> 매핑 객체가 리스트 카드와 상세 헤더에 중복되면 같이 갱신해야 한다. 카테고리 추가 시 `Record<InsightCategory, string>` 타입이 누락을 잡아준다.

## 유의사항

- **slug == id (1단계)**: 4단계 Supabase 이관 시 `id`를 UUID로 분리, 라우트에는 `slug`만 노출. 지금 분리 두면 마이그레이션 안전.
- **카테고리 enum 동기화**: 카테고리 추가 시 `insight-card.tsx`와 `app/insights/[slug]/page.tsx`의 `categoryTone`을 같이 갱신.
- **MDX 매핑 누락**: 새 인사이트 추가 시 `insightBodyMap`에 import+키 추가가 빠지면 fallback 텍스트로 떨어진다 (의도된 가드).
- **AI Timeline 색상 차용**: `--color-timeline-edit`(라벤더)는 DESIGN.md상 "제품 내 AI 액션 시각화 전용". 카테고리 뱃지에 차용하는 것은 **표면이 모임 사이트라 product가 아님**을 고려한 1회성 차용. 카테고리 색상이 더 정착되면 별도 `--color-category-workflow` 토큰으로 분리 고려.
- **readTime은 수동**: 자동 계산은 1단계 도입 X.
