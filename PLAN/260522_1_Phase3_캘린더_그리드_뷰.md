# Phase 3 — `/schedule` 월간 캘린더 그리드 뷰 추가

## Context

`/schedule` 페이지는 1단계 출시 이후 리스트 뷰만 제공해 왔다. 페이지 상단 안내문에 "캘린더 뷰는 추후 추가됩니다"가 명시되어 있고(`app/schedule/page.tsx:29-30`), CLAUDE.md의 3단계 잔여 작업에 "캘린더 그리드 뷰"가 포함된다.

목표는 한 페이지 안에서 **"이번 달 분포(그리드)"**와 **"일정 디테일(리스트)"**을 동시에 보여주는 것이다. 데이터는 2026-04 ~ 2026-07에 걸쳐 6건이 분포해 있어 단일 월 고정 뷰로는 절반 이상의 일정이 가려진다. 따라서 월 이동이 가능한 그리드가 필요하다.

이번 변경은 외부 캘린더 라이브러리를 도입하지 않고 `date-fns`(이미 설치, 한국어 로케일 포함)와 기존 디자인 토큰(`app/globals.css`)만으로 구현한다. DESIGN.md의 단일 액센트(Cursor Orange) 규칙과 80px 섹션 리듬을 그대로 따른다.

부수적으로, 현재 `ScheduleItemRow`의 `statusTone`은 "예정=오렌지 채움 / 진행중=semantic-success / 완료=메이티드"로 캘린더 결정과 어긋난다. 같은 페이지에 캘린더와 리스트가 공존하면 같은 일정이 다른 색으로 강조되어 일관성이 깨지므로, **리스트 측 색 매핑도 캘린더와 동일하게 통일**한다.

## 결정사항 요약 (인터뷰 확정)

1. **레이아웃**: `/schedule` 상단에 캘린더 그리드 섹션, 하단에 기존 리스트(다가오는/지난) 섹션. 토글/모달 없음.
2. **그리드 단위**: 월간 7×6(42칸 고정), **월요일 시작**. 초기 표시는 오늘 포함 월. 상단에 `< 2026.05 >` + "오늘" 컨트롤로 월 이동.
3. **클릭 동작**: 그리드 셀/일정 칩 클릭 → 하단 리스트의 동일 일정 카드로 `scrollIntoView({ behavior: 'smooth', block: 'center' })` + 1.4초 primary 펄스 하이라이트.
4. **반응형(단일 그리드)**: 데스크탑(md+) = 셀에 상태 색 칩 + 짧은 제목(최대 2개, 초과 시 `+N`). 모바일(md 미만) = 상태 색 도트만(최대 3개).
5. **상태 시각화(캘린더+리스트 통일)**:
   - 예정: `--color-ink` 1px 보더 + ink 텍스트(채움 없음)
   - 진행중: `--color-primary`(Cursor Orange) 채움 + `--color-on-primary` 텍스트
   - 완료: `--color-muted-soft` 텍스트 + `--color-hairline-soft` 보더 + opacity 60%
6. **인접 월 셀**: 옅게 표시(유지). 날짜 숫자 `--color-muted-soft`, 칩/도트 opacity 60%. 그리드 높이 안정성과 컨텍스트 연속성 확보.
7. **데이터 헬퍼**: `lib/content.ts` 신설하여 `getSchedule()`(start asc 정렬) 도입. CLAUDE.md 디렉토리 구조에 이미 명시되어 있다.
8. **새 색상 토큰/라이브러리**: 없음. 기존 토큰만 재조합.

## 핵심 구현 메모

### 데이터 흐름

```ts
// app/schedule/page.tsx (서버 컴포넌트)
const all = getSchedule(); // start asc
const upcoming = all.filter(s => s.status !== "완료");
const past = all.filter(s => s.status === "완료").reverse();

return (
  <>
    <HeaderSection />                          {/* 기존 80/64 패딩 */}
    <CalendarSection><ScheduleCalendar items={all} /></CalendarSection> {/* 64/64 */}
    <UpcomingListSection items={upcoming} />   {/* 64/64 */}
    <PastListSection items={past} />           {/* 64/80 */}
  </>
);
```

`ScheduleCalendar`만 client 경계. 리스트 카드는 서버 컴포넌트 유지하고 `id` props만 추가.

### `lib/date.ts` 확장 시그니처

```ts
export type CalendarDayCell = {
  date: Date;
  iso: string;            // 'yyyy-MM-dd'
  inCurrentMonth: boolean;
};

export function buildMonthGrid(
  monthDate: Date,
  opts?: { weekStartsOn?: 0 | 1 }
): CalendarDayCell[];     // 항상 42칸

export function formatMonthLabel(monthDate: Date): string;  // "2026.05"

export function indexScheduleByDay(items: ScheduleItem[]): Map<string, ScheduleItem[]>;
```

- `weekStartsOn` 기본값 1(월요일).
- 항상 6주(42칸) 반환 → 월 이동 시 그리드 높이 점프 없음.
- `isToday`는 클라이언트 effect에서 계산해 hydration mismatch 방지(서버에서 `new Date()` 직접 사용 금지).

### 하이라이트 구현 (방법 A — DOM 직접 토글)

```ts
// schedule-calendar.tsx 내부
function handleSelect(id: string) {
  const el = document.getElementById(`schedule-${id}`);
  if (!el) return;
  el.scrollIntoView({ behavior: "smooth", block: "center" });
  el.classList.add("is-highlight");
  if (timerRef.current) window.clearTimeout(timerRef.current);
  timerRef.current = window.setTimeout(() => {
    el.classList.remove("is-highlight");
  }, 1400);
}
useEffect(() => () => {
  if (timerRef.current) window.clearTimeout(timerRef.current);
}, []);
```

```css
/* app/globals.css */
@keyframes schedule-highlight-pulse {
  0%   { box-shadow: 0 0 0 0 var(--color-primary); }
  60%  { box-shadow: 0 0 0 3px var(--color-primary); }
  100% { box-shadow: 0 0 0 0 transparent; }
}
.is-highlight {
  animation: schedule-highlight-pulse 1.4s ease-out;
  border-radius: 8px;
}
```

> 1.4초 일회성 펄스는 영구 드롭 섀도우가 아니므로 DESIGN.md "드롭 섀도우 없음" 원칙을 해치지 않는다. 보수적으로 가야 하면 `outline: 1px solid var(--color-primary) + outline-offset: 2px` 1.4초 유지로 대체.

### `ScheduleStatusDot` 매핑 (단일 출처)

| 상태 | 칩 클래스 | 도트 클래스 |
|---|---|---|
| 예정 | `bg-[var(--color-canvas-soft)] text-[var(--color-ink)] border border-[var(--color-ink)]` | `bg-[var(--color-ink)]` |
| 진행중 | `bg-[var(--color-primary)] text-[var(--color-on-primary)] border border-transparent` | `bg-[var(--color-primary)]` |
| 완료 | `bg-transparent text-[var(--color-muted-soft)] border border-[var(--color-hairline-soft)] opacity-60` | `bg-[var(--color-muted-soft)]` |

리스트 카드의 상단 상태 뱃지도 같은 컴포넌트 재사용(`variant="chip"`).

### 셀 마크업 골격

```tsx
<div role="gridcell" aria-label="2026년 5월 8일 금요일, 일정 2건: ..."
     className="min-h-[64px] md:min-h-[112px] border-l border-t border-[var(--color-hairline)]
                p-2 cursor-pointer focus:outline focus:outline-1 focus:outline-[var(--color-primary)]"
     tabIndex={items.length > 0 ? 0 : -1}
     onClick={() => items[0] && onSelect(items[0].id)}
     onKeyDown={(e) => { if (e.key === "Enter" || e.key === " ") { e.preventDefault(); items[0] && onSelect(items[0].id); } }}>
  <div className="flex items-center gap-1">
    <span className={`code ${isToday ? "inline-flex w-6 h-6 items-center justify-center rounded-full border border-[var(--color-primary)]" : ""}`}>
      {format(date, "d")}
    </span>
  </div>
  {/* 데스크탑 */}
  <div className="hidden md:flex flex-col gap-1 mt-1">
    {items.slice(0, 2).map(i => <ScheduleStatusDot key={i.id} variant="chip" status={i.status} title={i.title} muted={!inCurrentMonth} />)}
    {items.length > 2 && <span className="caption text-[var(--color-muted)]">+{items.length - 2}</span>}
  </div>
  {/* 모바일 */}
  <div className="flex md:hidden gap-1 mt-1">
    {items.slice(0, 3).map(i => <ScheduleStatusDot key={i.id} variant="dot" status={i.status} muted={!inCurrentMonth} />)}
  </div>
</div>
```

### 그리드 헤어라인 처리

`grid grid-cols-7`에서 셀에 `border-l border-t`, 컨테이너에 `border-r border-b border-[var(--color-hairline)]` → 1px 라인 중복 없이 격자 완성. 모서리는 첫 행/열에서 자연스럽게 처리.

## 검증 방법

`pnpm dev` 후 다음 시나리오 확인:

**데스크탑 (≥1024px)**
- [ ] `/schedule` 진입 시 현재 월(예: 2026.05) 그리드가 보이고, 5/11과 4/27이 인접 월 셀에서 메이티드 톤으로 옅게 표시
- [ ] `>` 클릭 → 6월로 전환, 6/8·6/15·6/22 칩 노출. `<` 클릭 → 다시 5월. "오늘" 클릭 → 현재 월 복귀
- [ ] 6/8 "6월 정기 빌드 세션" 칩 클릭 시 하단 리스트의 동일 카드로 부드러운 스크롤 + 1.4초 오렌지 펄스
- [ ] 캘린더 칩과 리스트 상태 뱃지 색이 **동일하게** 표시(예정=잉크 보더, 진행중=오렌지, 완료=흐림)
- [ ] 오늘 날짜 셀의 숫자에 primary 1px 원형 인디케이터

**모바일 (<768px, devtools responsive)**
- [ ] 셀 안에 도트(최대 3개)만 표시, 칩/제목은 숨김
- [ ] 도트 또는 셀 탭 시 동일하게 리스트로 이동
- [ ] 날짜 숫자가 셀 좌상단에 잘리지 않고 표시

**접근성**
- [ ] Tab으로 prev/today/next → 그리드 셀 순회 가능, focus ring 보임
- [ ] Enter/Space로 일정 있는 셀 선택 시 리스트 스크롤 발동
- [ ] 월 이동 시 SR이 새 월 라벨 안내(`aria-live="polite"`)
- [ ] 일정 있는 셀의 aria-label에 날짜 + 일정 건수 + 제목 합성

**하이드레이션/성능**
- [ ] 콘솔에 hydration mismatch 경고 없음(`isToday` 클라이언트 계산 확인)
- [ ] 다른 페이지(`/`, `/notices`, `/projects`, `/contests`, `/about`)는 영향 없음

**빌드**
- [ ] `pnpm build` 통과
- [ ] Vercel Preview URL에서 동일 시나리오 재검증
