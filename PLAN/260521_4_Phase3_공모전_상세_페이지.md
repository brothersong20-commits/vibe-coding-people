# Phase 3 — 공모전 상세 페이지(`/contests/[slug]`) 구축

> 상태: ✅ 완료 (커밋 `eb8f1ec`)

## Context

홈페이지는 1·2단계까지 완료된 상태(라이브: https://vibe-coding-people.vercel.app/). 공모전 탭은 카드 리스트만 있고, 카드 우측 "공식 페이지 열기 ↗" 버튼이 사용자를 곧장 외부 공식 사이트로 보낸다. 모임 사이트로서 공모전을 단순 게시판이 아닌 **모임 차원의 도전판**으로 만들려면, 각 공모전마다 모임이 왜 주목했는지(모임 관점)와 누가 도전 중인지(팀/멤버)가 보여야 한다.

이번 메이저(Phase 3)는 그 두 가지를 담는 **공모전 상세 페이지**를 새로 만든다. 캘린더 그리드 뷰와 `/insights`는 별도 메이저(Phase 4, Phase 5)로 미룬다.

3단계 후보 세 개를 한 PR에 묶지 않고 하나씩 진행하기로 결정. 데이터 모델은 프로젝트가 이미 쓰고 있는 MDX 패턴(`content/projects/*.mdx` + `content/projects.ts` 인덱스)을 그대로 재사용.

## 결정사항

| 항목 | 결정 |

| 페이지 핵심 가치 | 외부 정보 요약 + 모임 관점(MDX) + "도전 중인 팀" 섹션 |
| 데이터 모델 | 완전 MDX 이관 — `content/contests/*.mdx`(frontmatter 사용 안 함, 본문만) + `content/contests.ts`는 인덱스/메타 모음 |

| 카드 동선 변경 | `ContestCard` 전체가 내부 `/contests/{id}`로 이동. 공식 외부 URL은 상세 페이지의 큰 "공식 페이지 보기" 버튼에서만 노출 |
| 빈 팀 상태 | "아직 등록된 도전 팀이 없습니다. Discord에서 팀 모집을 시작하세요." + (예정) Discord/오픈채팅 버튼 |
| URL 식별자 | 별도 slug 추가 없이 기존 `id`(예: `c-2026-mcst-ai-data`)를 URL 파라미터로 그대로 사용 |

## 데이터 모델

```ts
export type ContestTeamStatus = "모집중" | "확정" | "출품완료";

export type ContestTeam = {
  name: string;
  members: string[];
  status: ContestTeamStatus;
  note?: string;
  githubUrl?: string;
  demoUrl?: string;
};

export type Contest = {
  // 기존 필드 유지 (id, title, host, organizer?, startDate, endDate,
  //   prize, fields, eligibility, url, status, tags)
  summary?: string;
  teams: ContestTeam[];
};

export function getContestById(id: string): Contest | undefined;
```

## 페이지 사양 — `/contests/[slug]`

`app/projects/[slug]/page.tsx` 패턴 미러링. 4개 섹션:

1. **헤더** (max-w-960, padTop 80, padBottom 48) — 백 링크 + 상태 뱃지 + 태그 + 제목(`display-lg`) + 주최/주관 + summary + 메타 그리드(접수기간/시상/응모자격/분야) + **"공식 페이지 보기 ↗"** ink 버튼(외부, `target="_blank"`).
2. **모임 관점** (max-w-760, padTop 64, padBottom 80) — `prose-vcp` 안에 정적 import한 MDX `<Body />`. 정적 import 맵(`contestBodyMap`)으로 Turbopack/SSG 호환.
3. **도전 중인 팀** (max-w-1200) — `TeamCard` 그리드 또는 EmptyTeams 점선 카드 + Discord/오픈채팅 (예정) 버튼.
4. **다른 공모전** (max-w-1200) — 현재 항목 제외 + 마감 제외, 최대 3개 `ContestCard` 그리드.

### 메타데이터
- `generateStaticParams()` = `contests.map(c => ({ slug: c.id }))`
- `generateMetadata({ params })` = `${contest.title} — Vibe Coding People` / description = `summary ?? prize`.

## 카드 동선 변경

`components/cards/contest-card.tsx`:
- 전체를 `<Link href={\`/contests/\${contest.id}\`}>`로 래핑.
- 호버 `hover:bg-[var(--color-canvas-soft)]`.
- 외부 "공식 페이지 열기 ↗" 버튼 **제거** — 외부 이동은 상세 헤더로 일원화.
