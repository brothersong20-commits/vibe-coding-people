# Phase 1.0 커밋 + GitHub 푸시 + Vercel 연결 (2단계)

> 상태: ✅ 완료 (커밋 `8860df4`, `2578774`, `8e320fc`, `9a8a44f`)

## Context

Vibe Coding People 홈페이지의 1단계(로컬 개발)가 완성된 직후. 2단계(Git 초기화 + GitHub 푸시 + Vercel 자동 배포)로 넘어가는 메이저 전환점이다. 첫 커밋을 **Phase 1.0**으로 기록하고, 이후 커밋은 큰 변화는 `Phase 2`, `Phase 3` …, 소소한 업데이트는 `Phase 1.1`, `1.2` … 운영하기로 결정. 이 규칙은 `CLAUDE.md`에 영구 명시해 다음 세션부터도 일관 적용.

루트에 누적된 검증용 스크린샷(`vcp-*.png` 12개)과 `.playwright-mcp/`는 커밋 대상이 아니므로 삭제 + `.gitignore` 패턴 추가.

이번 단계의 산출물은 **로컬 git 저장소 + Phase 1.0 첫 커밋 + GitHub 푸시 + Vercel 자동 배포 연결**.

## 실행 순서 (실제 실행됨)

1. **CLAUDE.md 갱신** — "Git 커밋 규칙" 섹션 추가.
2. **임시 파일 정리** — 루트 `vcp-*.png`, `.playwright-mcp/` 삭제 + `.gitignore` 패턴 추가.
3. **Git 초기화 + Phase 1.0 첫 커밋** — `git init -b main` → 41개 파일 add → Phase 1.0 한국어 커밋(`8860df4`).
4. **GitHub 원격 연결 + 푸시** — 사용자 제공 URL(`brothersong20-commits/vibe-coding-people`)로 origin 추가 → `git push -u origin main`.
5. **Vercel 대시보드 연동** — 사용자가 vercel.com에서 Import Project로 GitHub repo 연결. Framework Preset Next.js 자동 감지, 환경 변수 없음. 배포 URL `https://vibe-coding-people.vercel.app/` 획득.
6. **`Phase 2` 커밋** — CLAUDE.md에 배포/저장소 섹션 + 단계 로드맵 상태 컬럼 추가, `.gitignore`에 `live-*.png` 패턴 추가 (`2578774`).
7. **`Phase 2.1` 커밋** — `app/icon.svg` 추가, 웜 크림 배경 + 잉크 'v' path (`8e320fc`).
8. **`Phase 2.2` 커밋** — `next.config.mjs`에 `/favicon.ico → /icon.svg` 308 redirect 추가, 콘솔 404 노이즈 완전 제거 (`9a8a44f`).
